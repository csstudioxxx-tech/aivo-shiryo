## 6. 資料ライブラリ（index.html）の仕様
- `index-library.html` を `index.html` として公開。ヘッダー＋H1「どこよりも、納得感のある仲介を。」＋検索ボックス＋`<div class="grid" id="grid">`（2列カード）＋`<div class="empty" id="empty">`＋フッター（管理者トグル）。
- **カードは3段**：一覧／詳細(`data-detail`)／編集(`data-edit-toggle`)。カード内にプレビュー・カルーセル（実スライドJPEG）、概要、収録内容、タグ(`data-tags`)、担当者別メモ(`data-memos`／STAFF=岡田/あきお/とうげん/田邊/横山/生沼)。検索は title＋概要＋収録＋tags＋memos を横断。
- **同時編集セーフ v2**（コンソール印 `Ai-voyage library build: concurrent-safe v2`）：1レコード1KV行（メモ `m:<res>:<id>`／タグ `t:<res>:<label>`／並び `order`）。`mergeRows`(upd大きい方優先)・`putRow`(=`if(!OWNER)return false` ガード)・KV経路 `GAS→localStorage('avlib:')→memory`・30秒サイレント自動更新。
- **ドラッグ並び替え**：オーナーのみ。`applyOrder()` が読み込み/更新時に並びを反映。**HTMLのカード順＝既定の並び順**（=`navy_frames.json` の順）。
- **管理者モード**：`CFG.OWNER_KEY:"aivo-okada"`（パスワード）。フッター「管理者として編集する」→入力で `body.owner` 付与＝編集解禁。非オーナーは閲覧のみ。`GAS_URL:""`（=未設定。各自ブラウザ保存。全員共有したいならGAS設定）。
- **カードを更新する手順（崩さない再生成）**：①`navy_frames.json` を作り直す（=各デッキのフレームを撮り直す）→ ②`<div class="grid" id="grid">` と `<div class="empty" id="empty"` の間だけを `card()` ビルダで差し替え → ③スクリプトとCSSは温存（再生成後に `concurrent-safe v2` / `applyOrder` / `OWNER_KEY` が残っているか必ず確認）。

`navy_frames.json` の各要素：`{id, href, title, about, sum, tags[], frames[base64 JPEG…]}`。`id→href` 対応：
```
houhou        → heya-sagashi-slides.html            私たちの部屋探しの方法
moushikomi    → heya-sagashi-bukken-moushikomi.html 業者サイトの物件と申込み5種類
kotsu         → heya-sagashi-kotsu.html             部屋探しのコツ
nyukyobi      → heya-sagashi-nyukyobi.html          入居日と、引越しの日取り
kimaranakatta → heya-sagashi-kimaranakatta.html     本日、物件が決まらなかった方へ
```
ライブラリのカード順（既定）＝この並び（=通話フロー順）。

## 11. 制作ワークフロー（次のClaude向け）
- 置き場：`/mnt/user-data/outputs/`。編集は Python で該当 `<section>` 内のみ置換。`<style>` のトークン・部品CSS・スクリプト・`<span class="g/r">` 位置は壊さない。
- 検証：Playwright(chromium, viewport 1280x720, device_scale_factor=2)でスクショ。撮影前に `.nav,.fsbtn,.hintbar{display:none!important}` を add_style_tag で隠す（deck-nextを見せたい時は隠さない）。`pageerror` を必ず確認（エラー0を担保）。
- プレビュー（ライブラリ用）：各スライドを撮り、PIL で 640x360・JPEG quality72・base64 にして `navy_frames.json` の `frames` に格納。
- **デッキを変更したら必ず**：①該当デッキのプレビュー撮り直し → ②`index.html` のカードを `card()` で差し替え（スクリプト/CSS温存を確認）→ ③`Ai-voyage-shiryo/` に各ファイルをコピー → ④`Ai-voyage-shiryo.zip` を作り直し → ⑤`present_files` で提示＋アップ手順を案内。
- 見せ方：**必ず `present_files` でスクショPNGを提示**（説明だけで済ませない）。デザイン候補は実物レンダリングで2〜3案出して選ばせる。

## 12. 新しい資料（デッキ）を追加する手順
1. 既存デッキ（例 `heya-sagashi-bukken-moushikomi.html`）を雛形にコピー → `<title>` と中身の `<section>` 群を差し替え。1枚目は `slide navy active`＋brandmark を維持。`#counter` の `1 / N` をページ数に合わせる。
2. 末尾の `deck-next`（資料一覧＋次の資料）を設定。前のデッキの「次の資料」リンクも更新。
3. `navy_frames.json` に `{id,href,title,about,sum,tags}` を追加し、frames を撮影。
4. ライブラリのカードを再生成（`card()`）。
5. プレビュー検証 → コピー → ZIP → 提示。
- **1スライド＝1メッセージ／図解多め／左右対比／最後はまとめ**を守る。
